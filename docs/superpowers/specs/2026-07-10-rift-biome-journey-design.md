# Rift Biome Journey — безшовний скрол крізь згенеровані біоми

**Дата:** 2026-07-10
**Статус:** затверджено до реалізації (slice 1)
**Гілка:** `feat/claude/rift-biome-journey`

## Проблема

Бій у Rift статичний: один ворог у центрі на фіксованому бекдропі, стадія просто росте. Немає відчуття руху крізь локації, як у Dungeon Crusher, де камера повільно веде вправо крізь одну велику горизонтальну локацію до наступних мобів, а біоми змінюються.

## Ціль (slice 1)

Позаду бою — **безшовний процедурний parallax-світ**, що плавно скролиться вправо на кожне вбивство моба, морфиться між біомами й ніколи не має швів/обривів. Плюс структура «біом = 10 стадій, бос на межі» і банер входу в біом.

## Ключова знахідка: структуру дає наявна каденція босів

Боси вже кожні 5 стадій (`boss.everyStages = 5`). Тож **без зміни балансу/двигуна**:
- Стадія, кратна 10 (10, 20, 30…) — **бос біому**: гейт у наступний біом + перехід.
- Стадія-бос, не кратна 10 (5, 15, 25…) — **міні-бос**: це вже наявний бій, лише інша тема/лейбл.
- Біом = 10 стадій; `biomeIndex(stage) = floor((stage - 1) / 10)`.

`STAGE_BANDS`, `GAME_BALANCE` і двигун **не чіпаються** — біом це чисто візуально-контентний шар, ключований від `stage`. Нуль колізії з балансом Codex.

## Не-цілі (окремі slice-и)

- Повноцінна унікальна бойова механіка міні-боса (таймер/поведінка) — у slice 1 лише візуальний маркер «Elite».
- Нові спрайти ворогів під біом (потребує арту) — реюз 3 наявних.
- Окремий екран-карта підземелля.

## Як досягається «без швів, одна велика локація»

Не тайли (там шви), а **процедурна неперервність**:
- Кожен parallax-шар — неперервна функція від світової координати. Силует рельєфу = сума кількох синусів різної частоти/фази (value-noise-подібно), детермінована від `worldX`. Камера їде вправо → семплюємо функцію на нових `x` → шва не буває **ніколи**.
- Пропси (кристали/спори) розставлені детерміновано по `x` (позиція з псевдо-random-бакета `floor(x / spacing)`), спавняться при вході в кадр і повертаються в пул при виході (патерн мого `shardPool`).
- Біоми **перетікають**: параметри функції (палітра, амплітуда/частота рельєфу, тип пропсів) інтерполюються протягом перехідної зони на межі біому → зміна біому без різкого стику.

**Тестовний інваріант «без швів»:** функція рельєфу Ліпшиць-обмежена — `|f(x+dx) − f(x)|` мала для малих `dx`, і неперервна на межі біому (кросфейд). Юніт-тест це перевіряє.

## Рух камери

- Бій: світ статичний, ворог у центрі. Бойова сцена (частинки, пул шардів, VFX) — **не чіпається**, перф збережено.
- Вбивство (`defeatSignal` інкремент): камера плавно (~0.8с, easeInOut) їде вправо на один сегмент (= одна стадія). **Рух несе світ** (скрол вправо — домінантна анімація «ми йдемо»). Ворог при цьому робить м'який fade-out і наступний **заходить справа в центр** (`beast.x` з +offset до 0) — це полиск поверх скролу, а не окрема «поїздка» ворога світом. Так відчуття руху дає безперервний паралакс, а не переміщення спрайта.

## Архітектура

### 1. `src/game/biome.ts` (нова, чиста, без Pixi)

```ts
export const BIOME_LENGTH = 10;
export type StageRole = 'mob' | 'mini-boss' | 'biome-boss';

export interface BiomeSpec {
  id: string;
  name: string;             // "Luminous Verge", "Ember Deep", …
  sky: [number, number];    // верх/низ градієнта неба
  terrain: number[];        // кольори шарів рельєфу (далекий→ближній)
  prop: number;             // колір пропсів
  amplitude: number;        // висота силуету
  frequency: number;        // щільність горбів
  propDensity: number;      // пропсів на одиницю x
}

export const BIOMES: readonly BiomeSpec[];             // 4–5 біомів
export const getBiomeIndexForStage: (stage: number) => number;
export const getBiomeForStage: (stage: number) => BiomeSpec; // wrap по BIOMES
export const getStageRole: (stage: number) => StageRole;     // %10==0→biome-boss; бос&%10!=0→mini-boss; інакше mob
export const terrainOffsetAt: (worldX: number, spec: BiomeSpec) => number; // неперервна сума синусів
export const propSlotAt: (index: number, spec: BiomeSpec) => { x: number; scale: number } | null; // детерм. розкладка
```

Уся логіка «який біом / яка роль / форма рельєфу» — чиста, юніт-тестується без GPU.

### 2. `src/views/RiftPixiScene.tsx` (адитивно)

Новий **персистентний** `world`-контейнер (найнижчий шар), створюється в app-init ефекті (deps `[]`), тож **не** руйнується при зміні ворога (на відміну від бойової сцени):
- Будує parallax-шари з `BiomeSpec` (небо-градієнт як заповнений прямокутник; 2–3 шари рельєфу як `Graphics`-полігони, перебудовані по семплах `terrainOffsetAt` у видимому вікні; пул пропсів).
- Ticker-callback: `cameraX` плавно тягнеться до `cameraTargetX` (інкремент на `defeatSignalRef` зміну); шари зсуваються на `−cameraX * parallaxFactor[шар]`; рельєф ресемплиться; біом-параметри читаються з `getBiomeForStage(stageRef)` і кросфейдяться при зміні; пропси спавн/пул за вікном.
- Деталізація (кількість шарів/пропсів) масштабується `renderProfile` (менше на `low`).
- Ворог: опційний м'який `enter-from-right` при білді сцени (beast.x з +offset до 0 за перші ~0.5с).
- Прибирання: `world.destroy({ children: true })` в app-cleanup; пул пропсів звільняється.

Експонує `data-biome={biomeId}` на host для e2e/спостережуваності (пишеться лише при зміні біому — не на кожен кадр).

### 3. `src/views/TheRift.tsx` (адитивно)

- Банер «Входимо в [Біом]» при зміні `getBiomeIndexForStage(stage)` — реюз патерну наявного `rift-clear-banner`.
- Маркер міні-боса: коли `getStageRole(stage) === 'mini-boss'`, лейбл «Elite» у `encounter-rank`.

### 4. Фон

Процедурний Pixi-світ стає фоном рифта; наявний CSS `--rift-backdrop-image` для rift-view знімається (щоб не було подвійного фону). Координація в `App.tsx`/`TheRift.css` — адитивно.

## Потік даних

```
defeat → stage++ (prop) → RiftPixiScene:
  defeatSignalRef змінився → cameraTargetX += segmentWidth
  ticker: cameraX →(ease) cameraTargetX; шари зсуваються; біом= getBiomeForStage(stageRef); кросфейд якщо змінився
  біом змінився → host.dataset.biome = id
TheRift: biomeIndex(stage) змінився → банер "Входимо в [Біом]"
```

## Перф

Parallax — кілька шарів `Graphics`, що зсуваються (transform, дешево); рельєф ресемплиться раз на кадр (кілька десятків точок полігону — дешево); пропси пуляться (нуль аллокацій у сталому режимі). Один твін cameraX на kill. Масштаб за render-профілем. Не додає per-tap роботи (світ рухається лише на defeat).

## Обробка помилок / крайові

- `prefers-reduced-motion` → камера стрибає без плавного скролу (миттєвий cameraX=target), пропси статичні.
- Дуже висока стадія → біом wrap по `BIOMES.length` (нескінченна подорож циклом біомів; палітри повторюються, але прогресія стадій триває).
- Точність координати: `terrainOffsetAt` семплить по `worldX mod PERIOD` (велика константа), щоб `sin` не втрачав точність на дуже великих `cameraX` (стадія в мільйонах) — рельєф лишається неперервним, бо `PERIOD` кратний періодам усіх синусів.
- WebGL недоступний → бойова сцена вже має фолбек; світ просто не рендериться (фон = суцільний колір неба з CSS-фолбеку).

## Тести

- **Юніт `biome.test.ts`:** межі `getBiomeIndexForStage` (1–10→0, 11–20→1, wrap); `getStageRole` (10,20→biome-boss; 5,15→mini-boss; 3,7→mob); **неперервність рельєфу** (`|terrainOffsetAt(x+ε) − terrainOffsetAt(x)|` обмежена для малих ε — інваріант «без швів»); детермінізм `propSlotAt`. Покриває нову поверхню (B5).
- **e2e:** сид на стадії біля межі біому (напр. 10) → вбивство → `data-biome` міняється й банер «Входимо в…» видимий. DOM-assert, не піксельний скрол.
- **Скріншоти при реалізації:** серія кадрів скролу + переходу біому, щоб звірити вигляд і безшовність (мій усталений патерн; не покладаюсь на опис).

## Режими / колізія

Працює в local і backend (світ керується `stage`/`defeatSignal`, що є в обох). Дотики: новий `biome.ts`, адитивні зміни `RiftPixiScene`/`TheRift`/`App`. `RiftPixiScene` Codex теж чіпає — мій світ у окремому персистентному контейнері, бойова сцена не переписується. Пуш рано.
